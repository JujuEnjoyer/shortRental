# ShortRental

Веб-приложение для управления краткосрочной арендой недвижимости. Проект объединяет в одном кабинете объекты, бронирования, календарь занятости, коммуникацию с гостями, шаблоны ответов, документы и гостевые ссылки для заполнения данных.

Проект разработан как выпускная квалификационная работа по теме "Веб-приложение для управления краткосрочной арендой недвижимости" и подготовлен к демонстрации как инженерный прототип.

## Основные возможности

- управление объектами недвижимости и бронированиями;
- импорт и экспорт календарей в формате ICS;
- единый чат с гостями через расширяемую интеграционную архитектуру;
- подключение платформ через адаптеры сообщений;
- шаблоны ответов с подстановкой данных гостя, объекта и ссылок;
- создание и редактирование шаблонов документов;
- гостевая ссылка для заполнения данных без регистрации;
- загрузка фотографий документа гостем;
- проверка данных гостя арендодателем;
- генерация итогового PDF-документа;
- аналитика по объектам, броням, сообщениям, документам и интеграциям;
- аудит значимых действий;
- политика хранения и ручная очистка устаревших персональных данных.

## Архитектура

Приложение построено на многоуровневой архитектуре:

- уровень представления: Thymeleaf, HTML, CSS, JavaScript;
- web-уровень: Spring MVC controllers;
- уровень безопасности: Spring Security, роли, guest-token, rate limiting, проверки владельца данных;
- сервисный уровень: бронирования, календарь, сообщения, документы, аналитика;
- интеграционный уровень: `ExternalMessagingPort`, `MessagingPortRegistry`, адаптеры платформ;
- уровень данных: Spring Data JPA, H2/PostgreSQL-ready модель, файловое хранилище.

Расширение интеграций выполняется через реализацию интерфейса `ExternalMessagingPort`. Благодаря этому новая платформа добавляется как отдельный адаптер, не меняя код чатов, шаблонов и документов.

## Безопасность и персональные данные

В дипломном прототипе реальные паспортные данные не используются. Для демонстрации применяются тестовые и обезличенные данные.

Реализованы прикладные меры защиты:

- разграничение доступа по роли арендодателя;
- проверка принадлежности объектов, гостей, документов и фото текущему арендодателю;
- временные гостевые ссылки;
- ограничение частоты запросов к публичным маршрутам и авторизации;
- безопасная обработка файлов: проверка расширения, имени файла и пути хранения;
- проверка внешних URL для ICS и HTTP endpoint-ов;
- журнал действий по операциям с гостями, документами и данными;
- политика хранения и ручная очистка старых паспортных данных, фото и PDF.

Для промышленного внедрения дополнительно требуется настроить HTTPS, reverse proxy/WAF, резервное копирование, аудит доступа на уровне инфраструктуры, регламенты обработки персональных данных и при необходимости шифрование чувствительных полей.

Подробнее: [docs/security-and-privacy.md](docs/security-and-privacy.md).

## Быстрый запуск

```bash
./mvnw spring-boot:run
```

После запуска приложение доступно по адресу:

```text
http://localhost:8080
```

Если порт `8080` занят:

```bash
./mvnw spring-boot:run -Dspring-boot.run.arguments=--server.port=8081
```

## Тесты

## Демо-сценарий

1. Зарегистрировать арендодателя.
2. Открыть личный кабинет.
3. Перейти в "Настройки" -> "Общие".
4. Нажать "Подготовить демо-сценарий".
5. Открыть гостевую ссылку в другом окне или инкогнито.
6. Заполнить тестовые данные гостя.
7. Вернуться в кабинет арендодателя, проверить данные и сформировать PDF.
8. Открыть "Настройки" -> "Безопасность" и посмотреть журнал действий.

Демо не отправляет настоящие сообщения реальным гостям и не требует настоящих паспортных данных.

Дополнительный чеклист: [docs/demo-checklist.md](docs/demo-checklist.md).

## Интеграции

В проекте используется единый интерфейс интеграции сообщений:

- Avito adapter;
- Cian adapter;
- Partner API adapters;
- Email Relay adapter;
- mock/fallback режимы для демонстрации платформ, где доступ к API ограничен партнерством.

Такой подход показывает готовность приложения к подключению разных каналов связи при наличии у пользователя API-ключей или партнерского доступа.

## Структура репозитория

```text
src/main/java/com/rental/shortrental
├── analytics      # сводная аналитика арендодателя
├── audit          # журнал значимых действий
├── booking        # бронирования и ICS-календари
├── document       # шаблоны, документы, PDF, гостевая ссылка
├── integration    # клиенты и адаптеры внешних платформ
├── messaging      # чаты, сообщения, шаблоны ответов
├── privacy        # политика хранения и очистка данных
├── property       # объекты недвижимости
├── security       # доступ, rate limit, проверка URL
└── user           # пользователи, гости, настройки профиля
```
